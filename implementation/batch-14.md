# Batch 14 - Detail Des Charges Completion

Status: completed

## Scope

Implemented the next level of `DETAIL DES CHARGES` parity so imported class 6 balances are no longer shown only as broad totals. The engine now creates detailed expense categories, account-level schedules, fiscal review markers, note links, income statement cross-links, and reconciliation output.

## Changes

- Expanded `ExpenseDetailsEngine` from broad class 6 buckets into detailed OHADA expense categories for purchases, stock variations, transport, external services, taxes, personnel, financial expenses, depreciation/provisions, and income tax expense.
- Added account-level expense lines under each category with N, N-1, variation, variation percentage, source account metadata, income statement line links, note links, and fiscal treatment.
- Preserved traceability through `source_account` references so each expense detail line can be drilled back to the imported account-level balance source.
- Added reconciliation fields comparing class 6 expense detail totals with the income statement expense lines generated by the calculation engine.
- Added validation coverage for expense details so validation reports a pass when totals reconcile, a critical issue when N totals diverge, and an N-1 warning when comparative expense totals diverge.
- Passed expense details into the validation pipeline from `FinancialStatementEngine`.
- Updated the charges detail screen data shape so the UI can consume the new note, fiscal treatment, and income statement cross-link fields.
